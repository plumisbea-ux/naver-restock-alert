# 카카오 알림톡 연동 및 개인정보 동의 메모

## 1. 카카오 알림톡 발송 구조

현재 MVP는 카카오 알림톡을 `mock` 발송 로그로 남깁니다. 실제 발송은 아래 조건이 준비된 뒤 붙입니다.

```text
1. 카카오톡 채널 개설
2. 비즈니스 채널 전환/인증
3. 알림톡 공식 딜러사 또는 발송 대행사 선택
4. 발신 프로필 등록
5. 재입고 안내 템플릿 등록 및 카카오 심사 승인
6. 딜러사 API KEY 발급
7. 서버 환경변수 등록
8. lib/mockSenders.js의 sendMockKakaoAlimtalk 부분을 실제 provider API 호출로 교체
```

카카오 알림톡은 자유문구 메시지가 아니라, 승인된 정보성 템플릿에 변수만 넣어서 보내는 구조로 설계해야 합니다.

## 2. 추천 템플릿 초안

```text
[재입고 안내]
신청하신 상품이 재입고되었습니다.

상품명: #{상품명}
옵션: #{옵션명}
대기인원: #{대기인원}명

재고는 조기 소진될 수 있습니다.
상품 확인: #{상품URL}
```

할인, 특가, 이벤트, 인기폭발, 놓치면 손해 같은 표현은 알림톡 템플릿에서 제외합니다.

## 3. 개인정보 수집 동의의 의미

톡톡에서 전화번호를 받는 목적은 카카오 알림톡 발송입니다. 따라서 최소한 아래 내용이 고객에게 고지되어야 합니다.

```text
수집 항목: 휴대전화번호
수집 목적: 신청 상품의 재입고 알림 발송
보유 및 이용 기간: 알림 발송 후 30일 또는 신청 취소 시 즉시 삭제
동의 거부 권리: 동의하지 않아도 톡톡 알림은 이용 가능
```

MVP의 버튼 동의는 “법적 효력을 완전히 보장하는 계약서”가 아니라, 실제 서비스에서 필요한 동의 UX와 로그 구조를 검증하기 위한 것입니다. 운영 전에는 개인정보처리방침, 동의 로그, 삭제/철회 절차, 위탁 처리 여부, 보관 기간, 접근 권한, 암호화/마스킹 처리를 함께 갖춰야 합니다.
